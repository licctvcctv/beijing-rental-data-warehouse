## 五、数据流程（详细）

整个项目的数据流经过 **10 个步骤**，从数据生成到最终可视化展示：

### 步骤 1：数据生成

- **脚本**: `/opt/rental-warehouse/crawler/gen_rental_data.py`
- **作用**: 使用 Python 生成 10000 条北京租房模拟数据
- **输出文件**: `rental_raw.csv`
- **数据覆盖**: 北京 16 个行政区、100+ 商圈
- **数据字段**: 房源ID、标题、类型（整租/合租/独栋）、行政区、商圈、经纬度、月租金、建筑面积、是否地铁房、装修情况、来源平台（链家/贝壳/58同城/安居客/自如）

```bash
# 执行数据生成
cd /opt/rental-warehouse
python3 crawler/gen_rental_data.py
```

### 步骤 2：上传 HDFS

- **脚本**: `/opt/rental-warehouse/scripts/run_upload.sh`
- **Java 类**: `com.beijing.wenyu.runner.UploadRunner`
- **作用**: 将 `rental_raw.csv` 上传到 HDFS
- **HDFS 目标路径**: `/data/logs/rental/rental_raw.csv`

```bash
# 执行上传
cd /opt/rental-warehouse
bash scripts/run_upload.sh
```

### 步骤 3：MapReduce 数据清洗

- **脚本**: `/opt/rental-warehouse/scripts/run_mr_clean.sh`
- **Java 类**: `com.beijing.wenyu.runner.EtlRunner` + `com.beijing.wenyu.etl.cleaner.CleanRentalJob`
- **作用**: 对原始数据进行清洗（去重、格式化、空值处理、字段标准化）
- **输入**: `/data/logs/rental/rental_raw.csv`
- **输出**: `/data/clean/rental/part-r-00000`（Tab 分隔的清洗后数据）

```bash
# 执行清洗
cd /opt/rental-warehouse
bash scripts/run_mr_clean.sh
```

### 步骤 4：Hive ODS 层（原始数据层）

- **脚本**: `/opt/rental-warehouse/scripts/run_hive_ods.sh`
- **SQL 文件**: `sql/hive/00_create_databases.sql` + `sql/hive/01_ods.sql`
- **作用**: 创建 4 个 Hive 数据库，创建 ODS 外部表 `ods_fy_jbxx` 指向 HDFS 清洗数据
- **表**: `rental_ods.ods_fy_jbxx`（EXTERNAL TABLE，直接读取 `/data/clean/rental` 目录下的数据）

```bash
# 执行 ODS 建表
cd /opt/rental-warehouse
bash scripts/run_hive_ods.sh
```

### 步骤 5：Hive DWD 层（明细宽表层）

- **脚本**: `/opt/rental-warehouse/scripts/run_hive_dwd.sh`
- **SQL 文件**: `sql/hive/02_dwd.sql`
- **作用**: 从 ODS 层读取数据，进行空值填充、单价计算，生成 Parquet 格式的明细宽表
- **表**: `rental_dwd.dwd_fy_mx`
- **核心计算**: `unit_dj = month_zj / jzmj`（租金单价 = 月租金 / 建筑面积）

```bash
# 执行 DWD 建表
cd /opt/rental-warehouse
bash scripts/run_hive_dwd.sh
```

### 步骤 6：Hive DWS 层（汇总层）

- **脚本**: `/opt/rental-warehouse/scripts/run_hive_dws_ads.sh`
- **SQL 文件**: `sql/hive/03_dws.sql`
- **作用**: 从 DWD 层按 6 个维度进行聚合汇总
- **输出**: 6 张汇总表（详见下方 Hive 分层表详情）

### 步骤 7：Hive ADS 层（应用层）

- **脚本**: 与步骤 6 同一脚本 `run_hive_dws_ads.sh`
- **SQL 文件**: `sql/hive/04_ads.sql`
- **作用**: 从 DWS 层提取直接面向 BI 可视化的应用数据
- **输出**: 8 张应用表

```bash
# 同时执行 DWS 和 ADS
cd /opt/rental-warehouse
bash scripts/run_hive_dws_ads.sh
```

### 步骤 8：Sqoop 导出到 MySQL

- **脚本**: `/opt/rental-warehouse/scripts/run_sqoop_export.sh`
- **作用**: 将 Hive ADS 层的 8 张表导出到 MySQL 的 `rental_result` 数据库
- **方式**: 先 TRUNCATE 目标表，再用 Sqoop export 写入数据

```bash
# 执行 Sqoop 导出
cd /opt/rental-warehouse
bash scripts/run_sqoop_export.sh
```

### 步骤 9：机器学习预测

- **脚本**: `/opt/rental-warehouse/scripts/run_prediction.py`
- **算法**: RandomForestRegressor（随机森林回归）
- **作用**: 基于建筑面积和行政区预测租金，结果写入 MySQL 的 3 张预测表
- **效果指标**: MAE = 1513 元, R2 = 0.6817

```bash
# 执行预测
cd /opt/rental-warehouse
python3 scripts/run_prediction.py
```

### 步骤 10：Superset 可视化

- **初始化脚本**: `/opt/rental-warehouse/superset/init_superset.py`
- **作用**: 自动创建数据库连接、11 张图表和 1 个仪表盘
- **访问**: http://192.168.6.167:8089 (admin/admin)

## 八、Hive 分层表详情

### 8.1 ODS 层（原始数据层）

**数据库**: `rental_ods`

#### 表: ods_fy_jbxx（北京租房房源基础信息原始表）

- **类型**: EXTERNAL TABLE（外部表）
- **存储**: TEXTFILE
- **分隔符**: Tab (`\t`)
- **HDFS 位置**: `/data/clean/rental`
- **字段数**: 13 个

| 序号 | 字段名 | 数据类型 | 说明 |
|------|--------|----------|------|

## 十、预测模块

### 10.1 基本信息

| 项目 | 详情 |
|------|------|
| 算法 | RandomForestRegressor（随机森林回归） |
| 库 | scikit-learn 1.3.2 |
| 脚本位置 | `/opt/rental-warehouse/scripts/run_prediction.py` |
| 数据来源 | MySQL `rental_result.ads_price_area_scatter` |
| 结果写入 | MySQL `rental_result` 的 3 张预测表 |

### 10.2 特征与目标

| 类型 | 内容 |
|------|------|
| 特征 1 | 建筑面积 (jzmj)，数值型 |
| 特征 2 | 行政区 (xzq)，LabelEncoder 编码为数值 |
| 目标变量 | 月租金 (month_zj) |

**注意**: 未使用 `unit_dj`（租金单价）作为特征，因为 `unit_dj = month_zj / jzmj`，使用会导致数据泄露。

### 10.3 模型参数

```python
RandomForestRegressor(
    n_estimators=100,    # 100 棵决策树
    max_depth=12,        # 最大深度 12
    random_state=42,     # 随机种子
    n_jobs=-1            # 使用全部 CPU 核心
)
```

### 10.4 模型效果

| 指标 | 值 | 说明 |
|------|-----|------|
| MAE | 1513 元 | 平均绝对误差，预测值与实际值平均偏差 1513 元 |
| R2 | 0.6817 | 决定系数，模型解释了 68.17% 的租金方差 |
| 测试集比例 | 20% | 80% 训练，20% 测试 |

### 10.5 输出结果

1. **ads_rent_predict_by_xzq**: 每个行政区的实际平均租金 vs 预测平均租金，以及偏差百分比
2. **ads_rent_feature_importance**: 两个特征（建筑面积、行政区）各自的重要性分数
3. **ads_rent_predict_scatter**: 从全量数据中随机抽样 500 条，记录实际租金和预测租金，用于散点图可视化

### 10.6 执行预测

```bash

python3 scripts/run_prediction.py
```

# nohup hive --service metastore &

### 常见问题排查

**问题 1: HDFS 安全模式导致写入失败**

```bash
# 报错: Cannot create file... Name node is in safe mode
# 解决: 退出安全模式
hdfs dfsadmin -safemode leave
```

**问题 2: Hive Metastore 未启动导致 Hive 查询失败**

```bash
# 报错: Could not connect to meta store
# 解决: 启动 Hive Metastore
systemctl start hive-metastore
# 等待 10 秒后重试
```

**问题 3: Sqoop 导出失败**

```bash
# 检查 MySQL 是否在运行
systemctl status mysqld

# 检查 rental_result 数据库是否存在
mysql -uroot -p123456 -e "SHOW DATABASES;"

# 如果 rental_result 不存在，重新创建
mysql -uroot -p123456 < /opt/rental-warehouse/warehouse/sql/mysql/01_wenyu_result.sql
```

**问题 4: Maven 编译失败**

```bash
# 进入 warehouse 目录重新编译
cd /opt/rental-warehouse/warehouse
mvn clean package -DskipTests
```

**问题 5: Superset 无法访问**

```bash
# 检查 Superset 服务状态
systemctl status superset

# 重启 Superset
systemctl restart superset

# 检查端口是否在监听
ss -tlnp | grep 8089
```
